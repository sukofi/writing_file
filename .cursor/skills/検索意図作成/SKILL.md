# 検索意図作成スキル

## 概要

`01.対策KW/[KWフォルダ]` 内の情報を読み、検索ユーザーのニーズを以下6項目で整理して Markdown に保存する。

- Who（誰の・想定読者）
- What（何を解決する・読者の検索意図）
- コンテンツのゴール
- ユーザーの最高の結果
- How（どうやって解決する）
- この記事でしか得られない情報

## 参照ファイル

基本は以下を優先して読む。

- `01.対策KW/[KWフォルダ]/③記事要項.csv`

`③記事要項.csv` がない場合は、以下を組み合わせて補完する。

- `01.対策KW/[KWフォルダ]/*AIO.csv`
- `01.対策KW/[KWフォルダ]/*②事前調査.csv`
- `01.対策KW/[KWフォルダ]/*①上位.csv`

## 手順

### 1. 対象KWフォルダを特定する

- ユーザーが `@01.対策KW` のように親フォルダだけ指定した場合は、配下のKWフォルダを確認して対象を特定する
- ファイル名の `[KW名]` には、必ず実際のKWフォルダ名を使う
- `検索意図：対策KW.md` のような仮名では保存しない

### 2. CSVから検索ニーズを読み取る

- `③記事要項.csv` があれば、Who / What / ゴール / How / この記事でしか得られない情報 の材料を優先取得する
- `AIO.csv` があれば、AI Overview の要約から検索結果の結論傾向を把握する
- `②事前調査.csv` から、顕在KW・潜在KW・PAA・QA・比較ニーズ・不安要素を拾う
- `①上位.csv` から、上位記事が共通で扱う論点、読者が気にしている比較軸、頻度・期間・向き不向きなどの主要論点を確認する

### 3. 6項目に整理する

各項目は抽象論で埋めず、CSVに出ているニーズをもとに整理する。

- **Who**: どんな悩みや前提を持つ読者か
- **What**: 読者が検索で知りたいこと、不安、比較したいこと
- **コンテンツのゴール**: 記事を読んだあとに何を理解させるか
- **ユーザーの最高の結果**: 読者が最終的にどう判断・行動できる状態が理想か
- **How**: どんな切り口、論点、比較、具体情報で解決するか
- **この記事でしか得られない情報**: 他記事との差別化要素、整理の切り口、一気通貫で得られる価値

### 4. Markdownで保存する

保存先とファイル名は以下で固定。

- `01.対策KW/[KWフォルダ]/検索意図：[KW名].md`

出力見出しは以下を使う。

```markdown
# 検索意図

## Who（誰の・想定読者）

## What（何を解決する・読者の検索意図）

## コンテンツのゴール

## ユーザーの最高の結果

## How（どうやって解決する）

## この記事でしか得られない情報
```

## 出力ルール

- ですます調は使わず、言い切りで統一する
- 「〜です」「〜ます」「〜状態です」は避ける
- 各項目は1〜2段落で簡潔にまとめる
- ふわっとした一般論ではなく、CSVで確認できた検索ニーズに寄せる
- `対策KW` という文字列をファイル名に残さない
- ユーザーが保存指定をした場合は、回答だけで終わらせず必ずファイル作成まで行う

## よくある論点

ピラティス系のKWでは、以下のような論点が出やすい。

- 本当に痩せるのか
- どれくらいの頻度・期間で変化するのか
- 体重減と見た目変化の違い
- マシンとマットの違い
- ヨガ・ジム・筋トレとの比較
- 向いている人、向いていない人
- 痩せない理由、続け方、食事との組み合わせ

## 仕上げ確認

- 対象KWフォルダ名をファイル名に反映したか
- 6項目がすべて入っているか
- 文末が言い切りになっているか
- CSVにない断定を足しすぎていないか
