# SEO記事・構成作成スキル

買取系SEO記事の「構成」を作成・最適化する。**参照・ルール**は「構成参照・ルール」スキルを参照。

## 実行手順

1. **01.対策KW** の該当KWフォルダを特定し、`③記事要項.csv`・`①上位.csv`（headline列）・`②事前調査.csv` を読む。
2. **02.レギュレーション** の「構成」表を確認（`レギュレーション.md`・`構成作成ルール.md`）。
3. **見出しを組み立てる**
   - 競合のトピックを MECE で網羅しつつ、記事要項の「この記事でしか得られない情報」をオリジナル見出しとして追加。
   - 「構成参照・ルール」の見出しルール（階層・結論ベース・KW・具体表現）に従う。
4. **見出しの順序を最適化**
   - 「構成参照・ルール」の標準順序（概要→相場→背景→高く売れる条件→…→まとめ）を目安に、検索ニーズの流れに沿って配置。
   - **買取CTAは訴求ブロック1つのみ**。h2「〇〇の査定・買取は『買取大吉』にお任せ」＋h3 店頭・出張・宅配。別の「売り方・買取方法」ブロックは設けない（重複回避）。
5. **構成ファイルに出力**
   - ファイル名：`構成：[KW名].md`
   - 保存先・フォーマットは「構成参照・ルール」に準拠。
   - **ファイル冒頭に「記事の方向性」セクションを必ず記載する**（下記フォーマット参照）。
   - 各 h2/h3 を `### h2：` / `- **h3**：` 形式で列挙。
   - リード文で含めること・表現メモ・ソースメモ・確認チェックリストを必要に応じて記載。
   - **見出し一覧（コピペ用）**を構成内に必ず含める：最後の h2（まとめ）の直後、「リード文で含めること」の直前に、見出し「## 見出し一覧（コピペ用）」を置き、その下に「左：hタグ、右：見出しタイトル（タブ区切り）」の説明と、コードブロック内に h1/h2/h3 と見出しタイトルをタブ区切りで列挙した一覧を記載する。
6. **QAブロックの見出し形式は必ず以下に統一する**
   - **h2**：`【Q&A】〜〜〜〜`（例：〇〇に関するよくある質問）
   - **h3**：`Q.〜〜〜〜`（質問文そのまま。各質問ごとに h3 で列挙）
   - 構成での表記例：
     ```
     ### h2：【Q&A】〇〇に関するよくある質問
     - **h3**：Q.〇〇は発行停止になったのですか？
     - **h3**：Q.〇〇が欲しい時はどうすればいいですか？
     ```

## 記事の方向性フォーマット

構成ファイルの冒頭（`# 構成案：〇〇` の直下）に以下のセクションを記載する。

```markdown
## 記事の方向性

### 想定読者
[検索意図・属性・悩みを1〜2文で記載]

### コンテンツのゴール
[記事を読んだあとに読者に理解・行動してほしいことを1〜2文で記載]

### 記事の流れ
[冒頭から結論・比較・注意点・訴求・Q&A・まとめまでの流れを2〜3文で簡潔に記載]

### トーン・言い切りの方向性
- [断定表現の可否・推奨する語尾など]
- [薬機法・医療法など法令上の注意がある場合は記載]

### クライアント訴求のポイント
- [訴求ポイント1]
- [訴求ポイント2]
- [訴求ポイント3]
```

## 見出し一覧（コピペ用）のフォーマット

構成ファイルに記載する際の形式：

```markdown
## 見出し一覧（コピペ用）

左：hタグ、右：見出しタイトル（タブ区切り）。以下をそのままコピーして利用できます。

\`\`\`
h1	[タイトル]
h2	[見出し]
h3	[見出し]
...
\`\`\`
```

ユーザーが「コピペ用だけチャットに」などと依頼した場合は、上記タブ区切り一覧のみをチャットに出力する。
