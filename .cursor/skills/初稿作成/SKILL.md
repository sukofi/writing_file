# 初稿作成・参照・流れ

買取系SEO記事の**初稿**（初稿.html）を作成・編集するときに参照するファイルと、見出し・内部リンクの扱い。初稿のmd出力は不要。

## 参照ファイル一覧

| 場所 | 用途 |
|------|------|
| `01.対策KW/[KWフォルダ]/構成：[KW名].md` | 見出しの出どころ。初稿の h1/h2/h3 はここに準拠する |
| `01.対策KW/[KWフォルダ]/ソース.md` | 数値・金額・根拠の引用元。初稿では金額・数値に必ずリンクを付ける |
| `01.対策KW/[KWフォルダ]/内部リンク候補.md` | 内部リンク候補（KW・URL・意図）。執筆・編集前に読み込む |
| `02.レギュレーション/レギュレーション.md` | リード・表現・まとめ・画像・装飾のルール |
| `02.レギュレーション/大吉訴求文.md` | 訴求ブロックの本文・CTA（h3で各買取方法を紹介、表は使わない） |
| `02.レギュレーション/記事編集ワークフロー-SKILL.md` | 内部リンク形式・語尾チェック・訴求の一元化・HTML/MD同期 |

## 冒頭のリード文の形

リード文は**「」で読者の考え・悩みを提示し、そのあとで記事がそれを解決することを紹介する**形で書く。

- **1段落目**：`「読者の悩み・考え1」` のみ。
- **2段落目**：`「読者の悩み・考え2」` のみ。
- **3段落目**：**補足文**（例：「このように考えていませんか？」「そんな疑問や不安を抱えている方も多いでしょう。」）。
- **4段落目以降**：この記事では〇〇を解説する・読めば〜できる、のように**それらを解決する記事であることを紹介**する。

並びは「」「」→ 補足文 → 「この記事では〜」の順。「」内は読者の心の声・疑問・悩みを具体的に書く。

## 初稿の見出し

- 初稿の**見出し**は、**構成ファイル（構成：[KW名].md）の見出し構成にそのまま準拠**する。
- h1（タイトル）・h2・h3 の文言・順序を構成ファイルから踏襲し、本文執筆時に崩さない。
- 初稿のファイル名：`初稿：[KW名].html`

## h2のリード分でh3を箇条書きにする（h3が存在する場合）

h3が存在するh2の場合、**h2のリード文で以下を必ず行う**。

1. **h3の見出しを箇条書き**で記載する（ul と li で列挙。箇条書きとh3見出しの文言は統一する）。
2. **その後に一言**を追加する（例：「それぞれ詳しく解説します。」「順に解説します。」「〇〇をまとめます。」）。

構造の例：
```
<h2>〇〇とは</h2>
<p>（導入文）</p>
<ul>
<li>（h3の見出し1）</li>
<li>（h3の見出し2）</li>
<li>（h3の見出し3）</li>
</ul>
<p>それぞれ詳しく解説します。</p>
<h3>（h3の見出し1）</h3>
...
```

## 金額・数値には必ず引用元リンクを付ける

初稿を作成する段階で、**金額や数値系の記述には必ず引用元をリンクでつける**。

- **対象**：買取相場・価格の目安、発行枚数、品位・重量・直径などのスペック、割合・率、その他具体的な数値。
- **やり方**：同一KWフォルダの **ソース.md**（またはファクトチェック・証拠一覧）を参照し、該当する数値・金額の根拠URLをアンカーテキストで設置する。外部リンクは `target="_blank"` `rel="noopener"` を付ける。
- **業者サイトのリンク**：根拠として設置してよいが、入稿前に削除するため **赤文字（`style="color:#c00;"`）** で設置する。詳細は **根拠リンク設置** スキルに従う。
- **ソースにない場合**：引用元がソース.mdに記載されていない数値・金額は、**ウェブ検索**で根拠となるページ（公式サイト・公的データ・信頼できる一次情報）を探し、**必ず**根拠リンクを設置する。リンクがどうしても見つからない場合のみ「目安」「業者・相場により異なります」などの注記を添える。

## 買取相場の注意書き（赤字で3か所に必ず記載）

記事内で買取相場などを紹介する初稿では、**以下の注意書きを必ず赤字で3か所に記載する**。

- **文言**：`※記事内で紹介している買取相場などは、買取市場での相場であり、『買取大吉』の買取価格を保証するものではありません。`
- **装飾**：`<p style="color:#c00;"><strong>※記事内で…</strong></p>` のように**赤字（`style="color:#c00;"`）・太字**で記載する。
- **設置箇所（いずれも必須）**：
  1. **記事冒頭**：リード段落の直後、「この記事でわかること」ボックスの前
  2. **相場の提示箇所**：買取価格・相場の見出し（h2）直下、相場の本文が始まる前
  3. **まとめ箇所**：まとめの見出し（h2）直下、まとめ本文の前

買取相場・価格の目安を扱う記事では初稿作成時に漏れなく追加する。

## 内部リンク候補.md を読み込む流れ

1. **事前に読み込む**：初稿の執筆または編集を始める前に、同一KWフォルダの **`内部リンク候補.md`** を読み込む。
2. **内容**：候補には KW テキスト・URL・意図（なぜそのリンクか）が含まれる想定。執筆・編集時に参照する。
3. **配置**：候補を**1か所にまとめず**、記事の流れ・各見出しの文脈に合わせて**分散配置**する（例：相場の説明の直後→相場系KW、売り方の直後→買取系KW）。
4. 形式・装飾は `02.レギュレーション/記事編集ワークフロー-SKILL.md` の「内部リンク」に従う（【関連記事】の下に KW・URL の2行。**KWはアンカーテキストにせず、リンクはURLに張る**）。

## 初稿完了後のh2画像生成

初稿HTMLが完成したら、**h2画像生成**スキルを発動する。各h2直下の画像を生成・圧縮・保存し、HTMLへ挿入する。詳細は `.cursor/skills/h2画像生成/SKILL.md` を参照。

## 他スキルとの役割分担

- **構成参照・ルール**：構成作成時の参照ファイル・見出しルール・構成ファイルのフォーマット。
- **構成作成**：構成ファイル（構成：[KW名].md）の作成・見出しの組み立てと出力。
- **初稿作成**（本スキル）：構成に沿った見出し＋内部リンク候補の読み込み流れ。**金額・数値には引用元リンクを付ける。** 編集時の細則は記事編集ワークフローへ。
- **根拠リンク設置**：数値・根拠へのリンクの付け方・業者サイトは赤文字で設置・入稿前に削除するルール。
