# 構成シートへの大吉訴求文ブロック追加

買取系SEO記事の構成（構成：[KW名].md）で、**訴求 → QA → まとめ** の順になるよう、レギュレーションの「大吉訴求文」ブロックを追加する。見出しフォーマットは「構成参照・ルール」に準拠。

## やること

1. **挿入位置・順序**  
   大吉訴求文（訴求）を **QAブロックの直前** に配置する。構成上の並びは **訴求 → QA → まとめ** とする。

2. **見出しの記載（構成シート形式）**  
   他の h2/h3 と同じ形式で、hタグと見出しタイトルを明記する。

   - **h2**：`ブランド品の査定・買取は『買取大吉』にお任せ`
   - **h3**：`店頭買取`
   - **h3**：`出張買取`
   - **h3**：`宅配買取`

3. **レギュレーション注記**  
   本文・CTA は `02.レギュレーション/大吉訴求文.md` の内容をそのまま挿入し、訴求・QA・まとめの順で配置する旨を1行で書く。**大吉訴求は表を使わず、h3で各買取方法を紹介する。**

## 構成ファイルへの追記テンプレート

```markdown
### h2：ブランド品の査定・買取は『買取大吉』にお任せ

- **h3**：店頭買取
- **h3**：出張買取
- **h3**：宅配買取

※**レギュレーション**：本文・CTAは `02.レギュレーション/大吉訴求文.md` の内容をそのまま挿入。訴求は表を使わずh3で紹介。訴求 → QA → まとめの順で、本ブロックはQAの直前に配置。

---

※PAA・QAで拾った質問を反映。（QAブロック）

---

### h2：まとめ：（既存のまとめ見出し）
```

## 注意点

- 大吉訴求文の**見出しタイトル**は、`02.レギュレーション/大吉訴求文.md` の h2/h3 と一致させる。
- 構成シートでは「hタグ＋見出しタイトル」を必ず設置し、本文は別ファイル参照でよい。
- 「戻して」と言われた場合は、削除されていた大吉訴求文ブロックを上記テンプレートで再度追加する。
