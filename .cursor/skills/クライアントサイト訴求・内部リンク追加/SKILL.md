# クライアントサイト訴求・内部リンク追加スキル

構成案（構成：[KW名].md）に対して、クライアントサイトの**訴求ブロック**と**内部リンク候補テーブル**を追加する。

## 関連ファイル

- `PADDLE_PILATES_コラム台帳.md`：`paddle-gym.com` 用の公開済みコラム台帳
- `サイト別コラム台帳テンプレート.md`：新規クライアントの台帳を作るときの雛形

## 事前確認

- ユーザーからクライアントサイトURLが渡されている場合は、**必ずそのURLのドメインを正として扱う**
- 別ドメインや旧サイトらしきURLを検索結果で見つけても、**ユーザーが指定したドメインを優先**する
- クライアントサイトURLが未共有なら、調査前にユーザーへ確認する
- 作業途中でユーザーから正しいURLへ修正が入った場合は、既に入れた参照URL・内部リンク候補・訴求メモ内のドメインを**すべて差し替える**

## 実行手順

### 1. クライアントサイトの調査

ユーザーから渡されたURLをもとに以下を収集する。

**サイト別コラム台帳がある場合**：
- まずスキル配下のサイト別コラム台帳を確認し、公開済みコラム候補の一次ソースとして使う
- `paddle-gym.com` の場合は、必ず `PADDLE_PILATES_コラム台帳.md` を先に読む
- 台帳に十分な記事候補がある場合、毎回WebSearchで一から探し直さなくてよい
- 台帳が古い、記事数が少ない、必要なテーマが不足している場合のみ、WebSearch・WebFetch・サイトマップで補完し、必要に応じて台帳を更新する

**訴求ポイントの収集**：
- WebSearchで `site:[ドメイン] column` などを検索し、コラム記事URLを一覧化
- `site:[ドメイン]` で広めに検索し、コラム以外の固定ページも内部リンク候補になりうるか確認
- サービスの特徴（立地・形式・専門性・体験メニュー・価格など）を把握
- WebFetchでサイトやコラムページの内容を取得（タイムアウト時はWebSearchで代替）

**台帳更新の優先手順**：
1. `robots.txt` や `sitemap.xml` を確認
2. 動的コラムサイトマップがあれば、そこから記事URLを抽出
3. 各記事のタイトルと概要を取得して、サイト別コラム台帳へ追記
4. 次回以降の内部リンク選定では、更新済み台帳を優先して参照

### 1.5 サイト別コラム台帳の保存ルール

- 台帳ファイルはスキル配下に置く
- ファイル名は `サイト名_コラム台帳.md` を基本とする
- まず `## 使い方` を置き、その下に運用方針を短く書く
- その後に `## 参照元`、`## 優先して使いやすい記事`、`## コラム記事一覧`、`## 運用メモ` を並べる
- `優先して使いやすい記事` は、実際に内部リンク候補へ転用しやすい記事を **5〜10本程度** に厳選する
- `コラム記事一覧` は、URL・タイトル・主なテーマ・使いやすい文脈を残す
- 新規クライアントで最初に台帳を作る場合は、`サイト別コラム台帳テンプレート.md` をベースにする
- 既存台帳がある場合は新規作成せず、追記・整理で対応する

**収集すべき情報**：
- スタジオ名・所在地・アクセス
- サービスの強み・専門性（他との差別化ポイント）
- 体験レッスンやトライアルの有無・内容
- コラム記事や固定ページのURL・タイトル一覧（理想は5本以上）

**公開済み記事が少ない場合の扱い**：
- 公開済み記事が少なくても、無理に別ドメインの記事を混ぜない
- 公開済みの内部リンク候補と、今後作成したい関連記事案を**分けて提示**する
- 公開済み候補が2〜4本程度でも、設置位置と導入文が自然なら採用してよい

### 2. 構成案に訴求ブロックを追加

**設置位置のルール**（レギュレーションの標準順序に準拠）：
```
…本文h2群 → 【訴求ブロック】 → 【Q&A】 → まとめ
```

**h2見出しの形式例**：
- `[エリア]で[テーマ]を始めるなら「[スタジオ名]」`
- `[テーマ]の悩みなら「[スタジオ名]」にご相談を`

**h3は3本構成**：
1. 専門性・指導内容の訴求（断定的な効果表現を避け「アプローチする」「サポートする」を使う）
2. 体験レッスン・トライアルの訴求（試しやすさ・判断しやすさを伝える）
3. 立地・空間・続けやすさの訴求

### 3. 執筆メモセクションを追加

訴求ブロックの直前（または見出し一覧の後）に `## [サイト名] 訴求ブロック 執筆メモ` を追加し、以下を記載する：
- 設置位置と訴求の方向性
- h3ごとの執筆ポイント（薬機法・健康増進法への注意事項を含む）
- 参照URLリスト（コラム記事URLと公式TOP）

### 4. 内部リンク候補テーブルを追加

`## 内部リンク候補（[サイト名] コラム記事）` セクションとして構成案に追記する。

内部リンク候補は **2種類** を対象とする：

#### A. クライアントサイトのコラム記事
WebSearch・WebFetchで収集した公開済みコラム記事や固定ページを対象とする。

- サイト別コラム台帳がある場合は、その中から**検索意図に近い順**に候補を選ぶ
- いきなり記事数を並べず、まず「優先して使いやすい記事」枠があればそこを確認する
- `paddle-gym.com` では `PADDLE_PILATES_コラム台帳.md` の「優先して使いやすい記事」から先に当てはめる
- 公開済み記事の候補は **3〜5本程度** を目安に厳選する

#### B. 同プロジェクト内の未公開記事（対策KW一覧）

以下のKWで作成中・公開予定の記事も内部リンク候補として分析する。
公開後にURLが確定するため、URL欄は「※公開後に設定」と記載する。

- 候補KWは広く確認してよいが、**実際に構成へ載せるのは関連度の高いものだけ**に絞る
- 目安は **3〜5本程度**。多くても6本までに抑える
- 検索意図の中心から外れる周辺KW（例：ニッチな属性、補助的な悩み、自宅トレだけで完結する話題）は無理に入れない
- すでにAの公開済みコラムで十分カバーできている論点は、Bで重複提案しすぎない

| 対策KW | 想定タイトル（例） | 推奨設置位置 | 導入テキスト例 |
|---|---|---|---|
| ピラティス 効果 | ピラティスの効果とは？ | h2「〇〇」h3「〇〇」末尾 | 「ピラティスの具体的な効果については、こちらをご覧ください。」 |
| ピラティス 初心者 | ピラティス初心者ガイド | 〃 | 〃 |
| マシンピラティス 効果 | マシンピラティスの効果 | 〃 | 〃 |
| ピラティスとヨガの違い | ピラティスとヨガの違いを解説 | 〃 | 〃 |
| ピラティス 頻度 | ピラティスの通う頻度 | 〃 | 〃 |
| ピラティス 痩せる | ピラティスで痩せる？ | 〃 | 〃 |
| ピラティス 向いてる人 | ピラティスが向いている人の特徴 | 〃 | 〃 |
| ピラティス 生理中 | 生理中のピラティス | 〃 | 〃 |
| ピラティス 男性 | 男性のピラティス | 〃 | 〃 |
| ピラティス 自宅 | 自宅でできるピラティス | 〃 | 〃 |

> ※ 上記は未公開記事のため、URL欄は「※公開後に設定」と記載し、公開後に差し替える。

**テーブル形式（出力時）**：

公開済み候補と未公開候補は、必要に応じて小見出しを分けて整理する。

| コラム記事タイトル | URL | 推奨設置位置 | 導入テキスト例 |
|---|---|---|---|
| 記事タイトル | https://... または ※公開後に設定 | h2「〇〇」h3「〇〇」末尾 | 「〇〇については、こちらをご覧ください。」 |

**内部リンク選定の基準**：
- 設置先のh3の内容と「話題が続く」記事を選ぶ（公開済み・未公開問わず）
- 読者が「もっと詳しく知りたい」と感じる自然なタイミングに設置
- 1つのh2ブロックに最大1〜2本まで（詰め込みすぎない）
- Bの未公開記事候補は「検索意図の中心にある論点」から優先し、数を厳選する
- 未公開記事は公開後にリンクを実装することを執筆メモに明記する
- 検索結果に別ドメインが混ざっても、ユーザー指定ドメイン以外は採用しない

**設置ルールを末尾に記載する**：
```markdown
**内部リンク設置ルール**：
- リンクはアンカーテキスト形式（`<a href="URL" target="_blank">テキスト</a>`）で設置
- 1つのh2ブロックに最大1〜2本まで
- 訴求ブロック内のリンクはテキストリンクを基本とし、最終版装飾時にblogcardへ変換を検討
- 競合サイトへのリンクは最終入稿時に必ず削除する
- 未公開記事へのリンクはURL未確定のため、公開後に差し替える
```

### 5. コピペ用見出し一覧を更新

構成内のコードブロック（コピペ用見出し一覧）に、追加した訴求h2/h3を漏れなく追記する。

## PADDLE運用例

- クライアントサイトURLが `https://paddle-gym.com/` の場合、まず `PADDLE_PILATES_コラム台帳.md` を読む
- Aの公開済みコラム候補は、`innermuscle`、`beautiful`、`yoga`、`triallesson`、`motorcontrol`、`experience` などから検索意図に応じて厳選する
- Bの未公開記事候補は、`ピラティス 効果`、`ピラティス 初心者`、`ピラティス 頻度`、`ピラティス 痩せる`、`ピラティスとヨガの違い` など中心論点を優先する
- `男性`、`生理中`、`自宅` などの周辺テーマは、検索意図の中心でない限り無理に入れない

- [ ] 薬機法に抵触する断定表現がないか

- [ ] Q&Aブロックが重複していないか（訴求追加時に重複しやすいため必ず確認）
- [ ] 未公開記事候補は検索意図に近い3〜5本程度へ厳選できているか
- [ ] 未公開記事のURL欄に「※公開後に設定」と記載されているか
